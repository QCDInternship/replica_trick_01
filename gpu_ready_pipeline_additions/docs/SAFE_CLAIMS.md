# Safe Claims

## Safe CV Bullet

Extended an SU(3) replica-lattice QCD codebase with a CPU-preserving
GPU-readiness pipeline, including profiling, bounded multiprocessing/MPI job
structure, optional CuPy-based measurement kernels, GPU-backend validation, and
experimental CUDA-generation scaffolding.

## Safe GitHub README Claim

This repository preserves the CPU generator as the authoritative physics path
while adding optional GPU measurement/action kernels, backend availability
checks, metadata-labelled validation gates, public-GPU benchmarking tools, and
isolated experimental CUDA-generation prototypes.

## Safe Project-Summary Paragraph

The project modernises a replica-lattice QCD workflow without replacing the
CPU physics reference. It adds CPU/HPC profiling and organisation, optional GPU
measurement kernels for saved configurations, readiness checks for future GPU
generation work, and public-GPU benchmark reporting. Experimental CUDA
generation code is isolated, labelled, and guarded by validation requirements.

## Unsafe Or Overstated Claims To Avoid

- GPU-accelerated SU(3) configuration generation.
- Full CUDA lattice-QCD simulation.
- Production-ready GPU Markov-chain sweeps.
- Physics-equivalent CUDA generator, unless statistically validated.
- GPU is the default backend.
- CuPy is required for normal CPU use.
